Similar to tracked_block_allocator, but shares the tracker with the higher level allocator.

This allows tracking both (de-)allocations and growth with one tracker.

Note
Due to implementation reasons, it cannot track growth and shrinking in the constructor/destructor of the higher level allocator.

Creates multiple nested binary_segregator.

If you pass one type, it must be a Then the result is a binary_segregator with that Segregatable and null_allocator as fallback. If you pass two types, the first one must be a Segregatable, the second one a Then the result is a simple binary_segregator with those arguments. If you pass more than one, the last one must be a RawAllocator all others Segregatable, the result is binary_segregator<Head, segregator<Tail...>>.

Note
It will result in an allocator that tries each Segregatable in the order specified using the last parameter as final fallback.
